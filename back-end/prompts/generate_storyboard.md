# 核心原则

## 1. 交互逻辑与因果状态原则（新增最高优先级）

在生成提示词时，必须清晰区分动作的方向、主体与客体，严禁模糊处理双向交互：

**动作方向性 (Action Direction)**：
- 攻击/受力：明确区分"打人"与"被打"。
  - 打人：描述施暴者的发力姿态、肌肉紧绷、拳头/武器挥出的轨迹、面部凶狠表情。
  - 被打：描述受害者的受力反应、身体失衡、头部偏转、痛苦表情、伤口显现过程。
- 给予/接收：明确区分"付款/给物"与"收款/接物"。
  - 付款/给物：描述手部伸出的动作、物品离开身体的轨迹、钱包/手机屏幕显示"支付成功"或余额减少、人物如释重负或吝啬的表情。
  - 收款/接物：描述手部承接的动作、物品进入掌控的状态、手机/POS机显示"到账"、人物贪婪或感激的表情。

**状态因果链 (Causal State)**：状态变化必须有视觉依据。例如：因为"被打"，所以"嘴角溢血"；因为"付款"，所以"手中钞票减少"或"手机界面变化"。

**人物状态 (Character State)**：外貌、衣着、情绪、生理状态（受伤/疲惫/兴奋）。需注明是谁的状态。

**场景状态 (Scene State)**：时间、天气、光线、环境破坏程度、氛围。

**物品状态 (Item State)**：新旧、破损、发光、位置、流向（如：钞票从左手递到右手）。

## 2. 对白铁律（最高优先级）

原文中的每一句对白必须逐字完整嵌入对应镜头的 videoPrompt（以及 description 摘要）。

格式为：角色名（附带当前状态描述）说："原文台词原话"。

严禁改写、缩写、意译、合并、跳过、总结。哪怕是语气词、助词、标点也必须一字不差。

严禁用"开口""说话"等描述替代台词内容。原文有几句对白，输出就必须有几句对白。

## 3. 100% 忠实原文

不得编造原文中不存在的情节、人物或物品。所有视觉描述必须基于原文隐含或明示的信息进行合理视觉转化。

## 4. 最小粒度拆分

每个镜头只描述一个连续不可分割的摄影机动作（时长约 2-8 秒）。

一旦动作发生本质变化、视角大幅切换或时间跳跃，必须切断为新镜头。

## 5. 首尾帧差异

firstFramePrompt 与 lastFramePrompt 必须体现镜头内的动态变化（动作进展、表情转换、镜头运动到位、光影变化、物品位置移动）。

## 6. 提示词自包含

每条提示词必须包含完整的场景、光线、角色外貌/状态、构图信息。不得依赖上下文，不得出现"同上"、"如前所述"、"保持上一镜头状态"等字眼。每个镜头都是独立的视觉孤岛。

## 7. 镜头运动必写

每个镜头必须明确描述摄影机的运动方式（推/拉/摇/移/跟/升/降/环绕/固定/手持晃动等）及其节奏。

## 8. 连贯性检查

相邻镜头之间必须逻辑连贯：前一个镜头的 lastFramePrompt 的结束状态，应能自然衔接下一个镜头的 firstFramePrompt 的起始状态。
